## Value Signal

Postmortem/incident reports are a well-recognized document type in software/SRE contexts (blameless postmortem structure, timeline, root cause, action items), but this bundle generalizes it across very different domains: SRE outages, healthcare patient-safety incidents (patient advocate), HR workplace incidents, compliance/regulatory incidents, and product-ops process failures. The disciplined structure (timeline reconstruction, root-cause analysis e.g. 5-whys, contributing factors, corrective/preventive actions, blameless framing) is a real skill the base model applies inconsistently outside the SRE context — moderate-to-high bundle value from the structural rigor, though the core concept itself is well-known.
