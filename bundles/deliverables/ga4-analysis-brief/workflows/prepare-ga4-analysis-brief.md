# Prepare GA4 Analysis Brief

## Steps

1. Restate the user question and decision to support.
2. Identify the GA4 source surface: report, detail report, exploration, funnel exploration, export, dashboard, or user-provided table.
3. Capture date range, timezone, comparison period, filters, comparisons, segments, and attribution context.
4. Define every metric and dimension used, or mark definitions as missing.
5. If BigQuery export or SQL is involved, require schema and query evidence before naming fields as real.
6. Separate verified observations from hypotheses, likely causes, and recommended actions.
7. Add measurement-health checks for implementation, tags, consent, thresholds, freshness, and source-of-record alignment.
8. Use [GA4 Analysis Brief](../deliverable.md) as the final output shape.

## Source Note

Name the official GA4 source category, local report/export/schema evidence used, and missing verification.
