# SEO Operating Principles

## Source Note Required

Every SEO diagnosis or recommendation should include a short source note covering:

- Official source category used, such as Google Search Central, Search Essentials, structured data documentation, or O*NET role taxonomy.
- User-provided evidence used, such as URLs, exports, screenshots, crawl files, logs, keyword reports, or analytics reports.
- Missing evidence needed before a hypothesis becomes a conclusion.

## Evidence Classes

- Verified from source: confirmed in official documentation or inspected user evidence.
- Provided by user: accepted as context but not independently verified.
- Assumed for draft: used only to structure an initial plan.
- Needs verification: cannot support a final recommendation yet.

## Reconciliation Discipline

When two SEO tools disagree, neither number is automatically right. Align source scope, date range, geography, device, query/page filters, canonicalization, sampling, bot filtering, attribution, and metric definition before choosing a working interpretation.

## Risk Controls

Recommendations that modify live systems need explicit confirmation. High-risk areas include robots directives, canonical tags, redirects, sitemap generation, structured data, internal linking at scale, CMS templates, tracking settings, and bulk content changes.
