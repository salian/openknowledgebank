# SEO Audit

## Inputs

- Site scope, target market, business goal, and priority page types.
- URL sample or crawl export.
- Search Console and analytics evidence when performance claims are requested.
- CMS/platform, robots, sitemap, canonical, redirect, structured data, and rendering evidence when technical claims are requested.

## Process

1. Define audit scope: pages, templates, market, device, date range, and search engine.
2. Separate technical accessibility, indexability, content relevance, internal linking, SERP competition, and measurement health.
3. Use [crawl-index-rank triage](../frameworks/crawl-index-rank-triage.md) to decide whether a problem is discovery, indexing, ranking, presentation, or measurement.
4. Classify each finding by evidence strength, expected impact, implementation risk, and reversibility.
5. Produce a prioritized [SEO audit brief](../deliverables/seo-audit-brief.md).

## Output Contract

The audit output should include a source note, evidence reviewed, findings, recommended actions, verification steps, and confirmation needed before risky changes.

If evidence is missing, the output should state what can be concluded now and what remains a hypothesis.
