# Content Optimization

## Inputs

- Existing page or draft.
- Target audience and conversion goal.
- Query or topic evidence with source, date, market, device, and tool if available.
- SERP observations or competitor examples, clearly marked as observed evidence.

## Process

1. Identify the user task behind the query or topic.
2. Map the page promise, sections, title, headings, internal links, and calls to action against that task.
3. Use [intent-to-page fit](../frameworks/intent-to-page-fit.md) to find gaps.
4. Recommend content changes that improve clarity, usefulness, and search-engine understanding.
5. Flag any claims, schema, reviews, ratings, prices, availability, credentials, or local details that need source verification.

## Output Contract

The final brief should separate verified evidence, assumptions, missing evidence, proposed edits, and measurement plan. It should not promise rankings or recommend manipulative repetition.
