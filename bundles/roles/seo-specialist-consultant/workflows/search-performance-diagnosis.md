# Search Performance Diagnosis

## Inputs

- Question being answered, such as traffic drop, ranking change, indexing issue, or conversion decline.
- Date range, comparison period, country, device, query/page filters, and source system.
- Search Console export, analytics report, rank tracker export, crawl evidence, or user-provided screenshots.

## Process

1. State which number is being evaluated and what it measures.
2. Align date ranges, filters, page/query scope, geography, device, attribution, and processing logic across tools.
3. Separate demand change, SERP change, measurement change, crawl/indexing issue, content relevance, technical implementation, and competitor effects.
4. Identify expected differences between tools before treating a discrepancy as a problem.
5. Recommend checks and actions in priority order.

## Output Contract

Answer the user directly, then show the evidence alignment. If the user asks "which number is right," state that neither number is authoritative until the definitions and settings are aligned, then identify the current best source for the specific decision.
