# Evidence-Led Design

For each recommendation, identify:

- Verified source facts: research findings, analytics definitions, product requirements, design-system rules, or implementation constraints the user supplied or the agent inspected.
- User-provided facts: details asserted by the user but not independently inspected.
- Assumptions: plausible design hypotheses that need validation.
- Needs verification: facts that must be checked before the recommendation becomes a decision.

This framework prevents the agent from presenting invented research, metrics, design-system details, or product constraints as settled facts.
