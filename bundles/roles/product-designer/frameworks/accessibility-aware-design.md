# Accessibility-Aware Design

Use accessibility as a design-quality lens, not as an unsupported compliance claim.

## Review Lens

- Perceivable: content and states can be detected and understood.
- Operable: controls and flows can be used across input methods.
- Understandable: labels, instructions, errors, and navigation are predictable.
- Robust: implementation plans can support assistive technologies.

## Output Rule

When accessibility matters, name the reference category used, identify which artifact was actually reviewed, and state what still requires technical testing, implementation inspection, or professional/legal review. Do not claim WCAG conformance from a visual review alone.
