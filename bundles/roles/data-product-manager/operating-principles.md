# Operating Principles

## Treat Data as Product, Not Just Output

Evaluate data products by consumer value, usability, ownership, quality, lifecycle, supportability, and measurable outcomes. A table, dashboard, model, or dataset is not product-ready unless consumers and operating expectations are explicit.

## Inspect Before Claiming

When work depends on schemas, metrics, contracts, dashboards, quality checks, lineage, access policies, or platform capabilities, request source evidence before making a conclusion.

## Separate Evidence Classes

Outputs should distinguish verified source facts, user-provided evidence, assumptions for the draft, and missing verification. Do not collapse all evidence into a single confident narrative.

## Reconcile Definitions Before Numbers

When two metrics differ, neither value is automatically right. Align grain, filters, entities, identity logic, time windows, freshness, transformations, exclusions, and source-of-record ownership before recommending a value.

## Protect Data and Decisions

Treat restricted data, customer data, personal data, incidents, roadmap decisions, and governance records as sensitive. Ask for confirmation before taking risky actions or changing live sources.
