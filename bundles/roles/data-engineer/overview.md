# Data Engineer

This bundle helps teams produce source-aware data engineering plans without turning plausible schemas or platform assumptions into confident implementation guidance.

## Source Discipline

- Start by identifying the source category: source schema, warehouse schema, API contract, data contract, orchestration config, transformation code, product analytics definition, access policy, or user-provided evidence.
- Treat user-provided diagrams, snippets, sample rows, and summaries as local evidence that may be incomplete or stale.
- Mark assumptions and missing evidence before recommending joins, pipeline logic, quality checks, scheduling, retention, permissions, or production rollout.

## Typical Covered Work

- Pipeline and ETL/ELT planning.
- Schema and warehouse modeling review.
- Data quality checks, lineage notes, and validation plans.
- Orchestration, freshness, backfill, and handoff planning.

## Review Gate

Route final reliance to the accountable data owner, platform owner, security/privacy reviewer, and production change reviewer before implementation.
