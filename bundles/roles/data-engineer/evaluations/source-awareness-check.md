# Data Engineering Source-Awareness Check

Use this rubric to review whether an answer follows the bundle.

## Checks

- Does it identify the data engineering decision and intended deliverable?
- Does it name source systems, schemas, contracts, code, samples, and local evidence used?
- Does it separate verified facts, user-provided claims, assumptions, and missing evidence?
- Does it define grain, keys, freshness, deduplication, lineage, and quality checks before implementation guidance?
- Does it require review and explicit confirmation before production, permission, export, schema, or infrastructure changes?

## Passing Standard

A passing answer is useful for data engineering planning while preserving source uncertainty, validation obligations, and production safety.
