# Source-Aware Finance System Triage

Use this workflow before producing requirements, configuration guidance, reporting specs, control notes, or implementation handoff.

## Steps

1. Confirm the decision: requirement, configuration change, report definition, integration, control evidence, testing plan, or rollout handoff.
2. Identify the evidence set: process documentation, ERP configuration, accounting policy, report definition, access model, integration spec, control evidence, tickets, and owner requirements.
3. Classify every important claim as verified from source, provided by user, assumed for draft, or missing.
4. Define business process, accounting/control impact, data source, permissions, environments, testing evidence, rollback path, and approval owners before recommending implementation.
5. Produce a plan with validation steps, open questions, review owners, and explicit confirmation before production changes.

## Output Contract

The final answer should include source scope, assumptions, functional requirements, control/accounting considerations, validation checks, risks, owner review, confirmation requirements, and a Source note.
