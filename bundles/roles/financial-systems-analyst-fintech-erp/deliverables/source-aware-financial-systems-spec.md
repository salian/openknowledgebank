# Source-Aware Financial Systems Specification

A financial systems specification should make process, configuration, accounting, and control assumptions explicit.

## Required Sections

- Objective, process scope, affected systems, and accountable owners.
- Evidence reviewed: configuration, process docs, accounting policy, report definitions, access model, integration specs, control evidence, and user context.
- Requirements and claims with status: verified, user-provided, assumed, or needs verification.
- Proposed configuration, reporting, workflow, integration, or control changes.
- Validation plan: test cases, reconciliation checks, access checks, control evidence, UAT owners, and rollback criteria.
- Review gates before production or financial-reporting reliance.

## Quality Bar

Do not present unverified system behavior, accounting treatment, permissions, controls, or report logic as facts. Include a Source note at the end.
