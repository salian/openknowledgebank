# API Change Brief

## Required Sections

- Direct readiness answer.
- Source note naming the API/spec/schema/docs/code evidence used.
- Change summary and non-goals.
- Affected consumers, integrations, environments, and support surfaces.
- Compatibility, versioning, migration, documentation, example, and SDK impact.
- Error handling, observability, security/privacy/accessibility/compliance, and rollback questions.
- Acceptance criteria and owner confirmations.

Do not state endpoint names, fields, status codes, rate limits, versioning policy, or compatibility guarantees unless the user provides or asks you to inspect the source evidence.
