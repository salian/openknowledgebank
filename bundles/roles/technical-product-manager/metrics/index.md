# Metrics

Technical product metrics vary by product and organization. Do not invent formulas or trends. Ask for the metric definition, owner, query/dashboard, source system, time window, filters, identity logic, aggregation, deduplication, and refresh cadence.

## Common Metric Classes

- User/customer outcome metrics: activation, retention, conversion, task success, adoption, satisfaction.
- Developer/platform metrics: API adoption, integration completion, time to first success, SDK usage, documentation task success.
- Reliability and operations metrics: availability, latency, error rate, incident frequency, support volume, rollback frequency.
- Delivery and readiness metrics: dependency completion, migration progress, test coverage evidence, launch gate status.

## Reconciliation Rule

When two tools disagree, neither value is automatically right. Define what each value measures, align scope, time period, filters, identity logic, aggregation, processing, and refresh timing, then state expected differences and unresolved discrepancies.
