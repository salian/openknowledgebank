# Evidence, Feasibility, Decision

Use this frame when a feature sounds desirable but is not yet ready for technical commitment.

| Layer | Question | Required Evidence |
| --- | --- | --- |
| Product evidence | What user or business problem is worth solving? | Customer evidence, analytics, support notes, sales context, strategy, or user research. |
| Technical feasibility | What must be true technically? | API/spec/schema/code/docs, service ownership, dependencies, capacity, risks, and owner input. |
| Decision | What can be decided now? | Owner, scope, non-goals, assumptions, confidence, risks, and next evidence request. |

The output should state what is verified, what is provided by the user, what is assumed, and what still needs verification.
