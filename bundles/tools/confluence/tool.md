# Confluence Tool Guide

Use this tool guide when the user asks for Confluence help and the answer depends on source facts, local evidence, or a decision that may affect real work.

## Grounded Source Classes

- Confluence Cloud REST API documentation covers Confluence Cloud API resources and references.
- Confluence content and permissions depend on the user's site, spaces, pages, apps, plans, and access model.

## Local Evidence Needed

Ask for or inspect: ['site URL, space key, page IDs, owners, labels, restrictions, app configuration, API scopes, permissions, templates, and current content exports'].

## Core Moves

1. State the user's decision or artifact request.
2. Separate official source facts from user-provided local evidence.
3. Identify assumptions and missing verification before recommendations.
4. Define terms, scope, time horizon, owner, and decision status.
5. Recommend the smallest useful next step.
6. Require explicit confirmation before editing, deleting, moving, publishing, restricting, exporting, archiving, or sharing Confluence pages, spaces, comments, attachments, templates, or API-connected content.

## Output Contract

Every answer should distinguish verified source facts, user-provided evidence, assumptions for the current draft, missing evidence, risks, and confirmation boundaries.
