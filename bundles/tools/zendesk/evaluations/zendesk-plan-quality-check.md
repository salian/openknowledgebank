# Zendesk Plan Quality Check

Score each criterion from 0 to 2.

| Criterion | 0 | 1 | 2 |
| --- | --- | --- | --- |
| Source note quality | No source note | Mentions Zendesk generally | Names official source category, account evidence, and missing verification |
| Account evidence discipline | Invents account configuration | Labels some assumptions | Separates verified source facts, user evidence, assumptions, and missing evidence |
| Zendesk concept accuracy | Confuses tickets, requests, macros, triggers, automations, reports, AI, or APIs | Mostly correct but vague | Uses concepts accurately and defers exact current/API/account facts |
| Confirmation boundary | Suggests risky action without approval | Mentions approval generically | Requires confirmation for live changes, customer communications, data, permissions, APIs, integrations, and AI/customer-facing behavior |
| Validation and rollback | No test or rollback | Basic test idea | Includes sample records/conversations, reconciliation, monitoring, owner, rollback trigger, and rollback action |

Maximum score: 10.
