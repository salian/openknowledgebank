# SAP S/4HANA Source Checks

## Official Source Categories

- SAP product pages for high-level positioning and deployment options.
- SAP Help Portal for edition-specific and release-specific product documentation.
- SAP Business Accelerator Hub for APIs, integration content, business events, and other integration assets.
- SAP Activate, implementation, feature-scope, release, security, and service documentation when relevant to a decision.

## Local Evidence Categories

- Tenant edition, release, localization, activated scope, licenses, and implementation constraints.
- Configuration, business roles, Fiori app access, approval workflows, master data, transactional samples, and custom extensions.
- Integration architecture, API contracts, communication arrangements, middleware, credentials ownership, logs, and error messages.
- Report definitions, CDS views, analytical queries, extracts, data warehouse models, reconciliation reports, and source-of-record rules.

## Do Not Invent

Do not invent SAP object names, table fields, CDS views, Fiori app IDs, business catalogs, configuration activities, scope items, APIs, business roles, migration objects, tax rules, statutory requirements, account assignments, or report formulas. Ask the user for evidence or direct them to inspect official SAP documentation and their authorized tenant.
