# DocuSign Source Checks

## Official Source Categories

- DocuSign product pages for high-level product naming and positioning.
- DocuSign Support Center for user-facing eSignature behavior, sending workflows, templates, envelope preparation, recipients, fields, signing order, account settings, and administrator-facing guidance.
- DocuSign Developer Center and eSignature REST API documentation for API, authentication, envelope, template, recipient, tabs/fields, status, embedded signing, Connect/webhook, and integration behavior when authorized programmatic access is in scope.
- DocuSign legal, trust, commercial, release, pricing, plan, and product availability pages when current legal/commercial terms, limits, editions, availability, or release-specific behavior matters.

## Local DocuSign Evidence

Ask the user for the relevant local evidence before final claims:

- account, edition/product, user, group, permission, brand, notification, identity/authentication, and enabled-feature evidence;
- template, envelope, document, recipient, role, routing/signing-order, tab/field, conditional logic, reminder, expiration, and sender evidence;
- status, report, completion certificate, audit trail, dashboard, export, and reconciliation evidence;
- integration key or app description, OAuth/JWT flow, scopes, environment, callback URL, Connect/webhook event selection, payload mapping, logs, monitoring, and sandbox/demo test evidence;
- legal, compliance, privacy, security, retention, data-classification, and records-management requirements.

## Claims That Need Verification

Do not make authoritative claims about these without current source or local evidence:

- account IDs, template IDs, envelope IDs, document names, recipient names, recipient emails, roles, routing order, signing groups, tab/field types, field locations, conditional logic, brands, reminders, expirations, notification settings, authentication methods, or permission profiles;
- whether a template, envelope, integration, webhook, embedded signing flow, report, export, completion certificate, or audit trail is complete, current, production-ready, compliant, or legally sufficient;
- API endpoints, payload fields, OAuth/JWT details, scopes, Connect/webhook events, callback behavior, rate limits, feature limits, product availability, plan/edition entitlements, release-specific behavior, or enterprise/admin settings;
- legal enforceability, consent, identity proofing, retention, privacy, cross-border transfer, regulated-industry, or records-management conclusions.
