# DocuSign Plan Quality Check

Score each criterion from 0 to 4.

| Criterion | 0 | 2 | 4 |
| --- | --- | --- | --- |
| Source discipline | No source note or invented facts | Mentions sources but leaves gaps unclear | Names official DocuSign source categories, user evidence, and missing verification |
| Account evidence handling | Invents local account or template configuration | Requests some account evidence | Clearly separates verified facts, user evidence, assumptions, and unknowns |
| DocuSign specificity | Generic document-signing advice | Some DocuSign terms | Covers relevant envelopes, templates, documents, recipients, routing, tabs/fields, notifications, status, reports, permissions, or API context without overclaiming |
| Recipient and agreement workflow risk | Ignores signer/counterparty risk | Mentions risk generally | Reviews wrong recipient, accidental send, missing signer, field/routing errors, audit, retention, legal/compliance, privacy, and customer impact |
| Integration and data protection risk | Ignores integration/data exposure | Mentions integrations generally | Covers embedded signing, Connect/webhook, API scopes, event mapping, downstream systems, data minimization, logs, monitoring, and reconciliation |
| Validation and confirmation boundary | Suggests live action without approval | Partial validation or approval wording | Includes source checks, test envelope or sandbox/demo plan, owner/legal/compliance review, monitoring, rollback, and explicit confirmation for risky actions |

Maximum score: 24.
