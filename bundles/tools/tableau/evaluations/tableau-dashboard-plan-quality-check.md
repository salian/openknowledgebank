# Tableau Dashboard Plan Quality Check

Score each criterion from 0 to 4.

| Criterion | 0 | 2 | 4 |
| --- | --- | --- | --- |
| Source discipline | No source note or invented facts | Mentions sources but leaves gaps unclear | Names official Tableau source categories, user evidence, and missing verification |
| Local evidence handling | Invents workbook/site/data-source configuration | Requests some local evidence | Clearly separates verified facts, user evidence, assumptions, and unknowns |
| Tableau specificity | Generic dashboard advice | Some Tableau terms | Covers relevant workbook, dashboard, data source, relationship, join, union, extract, refresh, permission, or publishing context without overclaiming |
| Dashboard and data-source logic | Ignores metric definitions and grain | Partial dashboard/data model plan | Covers audience, questions, metrics, dimensions, filters, grain, calculations, source-of-record, freshness, and reconciliation |
| Publishing and permission risk | Ignores access/data exposure | Mentions access risk generally | Reviews project, owner, groups/users, capabilities, embedded credentials, exports/downloads, subscriptions, row-level security, and sharing scope |
| Validation and confirmation boundary | Suggests live action without approval | Partial validation or approval wording | Includes source reconciliation, sample checks, permission tests, refresh checks, accessibility review, rollback, and explicit confirmation for risky actions |

Maximum score: 24.
