# Tableau Source Checks

## Official Source Categories

- Salesforce Tableau product pages for high-level product naming and positioning.
- Tableau Help for connecting to data, data sources, relationships, joins, unions, data blending, extracts, calculations, dashboards, publishing, permissions, subscriptions, alerts, downloads, accessibility, and Tableau Prep behavior.
- Tableau Cloud, Tableau Server, and Tableau Desktop documentation for environment-specific behavior.
- Tableau release notes, pricing, packaging, connector, API, and developer documentation when current feature availability, commercial terms, connector behavior, API syntax, limits, or release-specific behavior matters.

## Local Tableau Evidence

Ask the user for the relevant local evidence before final claims:

- workbook, worksheet, dashboard, story, view, owner, project, site, and environment evidence;
- data source, published data source, source system, connection, schema, field, calculated field, parameter, filter, relationship, join, union, blend, extract, live connection, refresh, and data-quality evidence;
- metric definition, source-of-record, grain, aggregation, level of detail, date/timezone, row-level security, and data freshness evidence;
- group/user, role, capability, permission, embedded credential, download/export, subscription, alert, embed, and sharing evidence;
- privacy, security, accessibility, retention, compliance, governance, approval, and rollback evidence.

## Claims That Need Verification

Do not make authoritative claims about these without current source or local evidence:

- workbook, dashboard, worksheet, view, field, calculated field, parameter, filter, data source, extract, refresh, or schedule names;
- whether a relationship, join, union, blend, calculation, or filter preserves the intended grain and metric definition;
- dashboard numbers, variance explanations, conversion rates, financial figures, forecasts, or operational KPIs;
- project, group/user, permission, capability, embedded credential, export/download, subscription, alert, or sharing behavior;
- connector availability, license requirements, pricing, limits, API endpoints, release-specific behavior, or feature availability.
