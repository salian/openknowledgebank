# Plan Funnel Analysis

1. Define the funnel decision and expected action.
2. Request each funnel step's event name, required properties, exclusion logic, order rules, time window, segment, platform, and environment.
3. Verify identity logic, timezone, date range, release window, bot/internal traffic handling, and data freshness.
4. Decide whether cohorts, experiments, or session replay evidence are needed to explain the result.
5. State what the funnel can show and what it cannot prove.
6. Recommend validation against source-of-record data or independent instrumentation when the decision is high impact.
