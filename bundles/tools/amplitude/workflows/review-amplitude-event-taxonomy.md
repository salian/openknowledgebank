# Review Amplitude Event Taxonomy

1. State the product question and decisions the taxonomy must support.
2. Request the relevant event list, property dictionary, identity rules, environment scope, owner, and instrumentation source.
3. Identify ambiguous events, duplicate meanings, missing properties, inconsistent naming, stale instrumentation, and environment leakage.
4. Check whether chart definitions use the intended event/property logic.
5. Separate taxonomy findings from product conclusions.
6. Require confirmation before recommending instrumentation, schema, governance, or dashboard changes.
