# HubSpot Sales Hub Source Checks

## Official Source Categories

- HubSpot product pages for high-level product positioning and naming.
- HubSpot Knowledge Base for Sales Hub, CRM, sales workspace, sequences, meetings, forecast, reports, permissions, workflows, commerce, and admin behavior.
- HubSpot Developers documentation for CRM objects, records, properties, associations, APIs, app authorization, and date-versioned API behavior.
- HubSpot legal, product catalog, pricing, release, and contract evidence when edition, seat, entitlement, add-on, commercial availability, or limit claims matter.

## Local Portal Evidence

Ask the user for the relevant local evidence before final claims:

- portal, edition, seat, add-on, billing, enabled-feature, team, owner, and permission evidence;
- object, record, property, association, pipeline, stage, lead status, lifecycle stage, list, view, import, and data-quality evidence;
- sequence, template, email, meeting, task, playbook, workflow, enrollment, suppression, unsubscribe, consent, and deliverability evidence;
- report, dashboard, forecast, goal, owner/team scope, filter, date field, currency, attribution, and data-refresh evidence;
- integration user, private app, OAuth app, API version, scope, endpoint, payload, sync, limit, and audit evidence.

## Claims That Need Verification

Do not make authoritative claims about these without current source or local evidence:

- custom object, property, internal name, pipeline, stage, lead status, lifecycle stage, list, owner, team, report, dashboard, workflow, sequence, or permission names;
- report numbers, forecast numbers, activity counts, conversion rates, deal velocity, source attribution, revenue, close dates, stage probability, or goal attainment;
- edition availability, seat requirements, pricing, add-ons, limits, beta availability, AI feature availability, or release-specific behavior;
- API endpoint syntax, payloads, scopes, rate limits, app authorization, or integration behavior;
- whether a workflow, sequence, outreach step, quote/payment behavior, import, export, integration, or API action is safe to run.
