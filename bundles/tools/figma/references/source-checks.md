# Figma Source Checks

## Official Source Categories

- Figma product pages for high-level product naming and positioning.
- Figma Help for Figma Design file behavior, layers, frames, components, styles, variables, libraries, prototypes, comments, Dev Mode, sharing, permissions, exports, and accessibility behavior.
- Figma developer documentation for API and integration behavior when authorized programmatic access is in scope.
- Figma release, pricing, plan, plugin, and beta-feature documentation when current feature availability, commercial terms, integration behavior, limits, or release-specific behavior matters.

## Local Figma Evidence

Ask the user for the relevant local evidence before final claims:

- file, project, team, page, frame, section, layer, branch, version, owner, and collaborator evidence;
- component, instance, variant, style, variable, library, token, icon, asset, documentation, and description evidence;
- prototype flow, interaction, animation, overlay, device, accessibility, content, and usability evidence;
- Dev Mode, inspect, measurement, asset export, code reference, component mapping, implementation, and repository evidence;
- sharing, link access, permissions, ownership, comments, notifications, exports, privacy, security, and approval evidence.

## Claims That Need Verification

Do not make authoritative claims about these without current source or local evidence:

- file, page, frame, section, layer, component, instance, variant, style, variable, library, token, prototype, comment, owner, collaborator, or permission names;
- whether a component, variable, style, token, prototype flow, or Dev Mode view is complete, current, accessible, or implementation-ready;
- whether an implementation matches the Figma design, design system, responsive behavior, accessibility expectation, asset export, or code mapping;
- whether sharing, link access, comments, exports, plugins, integrations, API access, or permissions are safe for the intended audience;
- pricing, plan limits, beta availability, plugin behavior, API endpoints, rate limits, MCP behavior, release-specific behavior, or enterprise/admin settings.
