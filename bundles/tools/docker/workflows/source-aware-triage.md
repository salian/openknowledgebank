# Docker Source-Aware Triage

1. Identify the scenario, organization, user goal, workflow, system, artifact, decision, or population at issue.
2. Name the official or authoritative source category and inspect current source text before relying on exact claims.
3. Separate verified source facts from local evidence and assumptions.
4. Ask for missing policies, configuration, records, examples, approvals, exports, source text, or reviewer decisions.
5. Produce a short triage answer with a confidence note and qualified-review gate.

## Output Format

- Source note
- Known facts
- Missing evidence
- Applicability analysis
- Risks and caveats
- Next checks
- Qualified reviewer
