# Linear Evidence Review

1. Identify the user's decision and audience.
2. List the official source categories relevant to the question.
3. Request local evidence: ['workspace, team, issue, project, cycle, label, status, priority, assignee, estimate, dependency, automation, Git integration, and API permission evidence'].
4. Classify each claim as verified, provided, assumed, or missing.
5. Note risks, dependencies, owners, and confirmation boundaries.
6. Produce a concise recommendation with evidence gaps.
