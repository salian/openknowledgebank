# Jira Source Checks

## Official Source Categories

- Atlassian Jira product pages for high-level product naming and positioning.
- Jira Software Cloud Support for boards, backlogs, sprints, reports, workflows, statuses, transitions, work items/issues, and project/space behavior.
- Jira Cloud Administration Support for site, permission, audit, user, group, product, and administration behavior when relevant.
- Jira Automation Support for triggers, conditions, branches, actions, smart values, permissions, and notification side effects.
- Atlassian Developer Jira Cloud REST API documentation for authorized API behavior, resource categories, scopes, pagination, payloads, and current endpoint details.

## Local Jira Evidence

Ask the user for the relevant local evidence before final claims:

- Jira site, product, Cloud/Data Center scope, plan, apps, project/space, board, filter, backlog, sprint, version, component, dashboard, and report settings.
- Work item/issue type, key examples, fields, custom fields, statuses, transitions, workflow rules, resolution logic, labels, assignees, teams, owners, and permissions.
- JQL, field/function availability, app-provided functions, saved filters, quick filters, and dashboard gadgets.
- Automation rule configuration, rule owner, audit/history, triggers, conditions, branches, actions, smart values, and notification impacts.
- REST API endpoint, scope, authorization context, payload, pagination, read/write mode, rate/limit concern, data classification, and dry-run/rollback option.

## Claims That Need Verification

Do not make authoritative claims about these without current source or local evidence:

- local project/space keys, boards, filters, columns, swimlanes, statuses, transitions, workflows, required fields, custom fields, work types, permissions, users, groups, or automations;
- whether a work item belongs on a board, backlog, sprint, version, release, epic, component, team, or report;
- whether a JQL query is executable in the user's Jira site;
- whether a report total, board count, export, dashboard value, or API response is the correct value until filters, permissions, status/resolution logic, time windows, and aggregation are aligned;
- pricing, plan limits, Marketplace app behavior, Rovo or agentic features, Data Center behavior, API endpoint details, scopes, and release-specific behavior.
