# Define Funnel

## Steps

1. State the business decision and conversion outcome.
2. List the intended journey steps in order.
3. For each step, require evidence for the event, dimension, page, screen, CRM status, or warehouse field that defines completion.
4. State whether the funnel is open, closed, or tool-specific equivalent.
5. State whether each step must directly follow the previous step, may indirectly follow it, or must happen within a time window.
6. Choose counting unit and denominator basis.
7. Record date range, timezone, filters, attribution, and reporting window.
8. Mark unverified step names and conditions as assumptions or missing evidence.

## Output

Use the definition section of [Funnel Analysis Brief](../deliverables/funnel-analysis-brief.md).

## Source Note

Name the official analytics documentation category, the report or schema evidence supplied by the user, and any missing step, denominator, or scope evidence.
