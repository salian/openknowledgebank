# Agile Alignment Review

## Steps

1. State the review question and the evidence supplied.
2. Map the evidence to Agile value and principle themes.
3. Identify strong signals, weak signals, contradictions, and unknowns.
4. Separate method compliance from Agile outcomes.
5. Check whether working software, customer feedback, technical excellence, sustainable pace, simplicity, and adaptation are visible.
6. Recommend small experiments or evidence requests before broad process changes.
7. Mark live workflow, staffing, reporting, contract, or customer-facing changes as requiring confirmation.

## Output

Use [Agile Operating Brief](../deliverables/agile-operating-brief.md).

## Source Note

Name the Agile framework source, the local artifacts supplied by the user, and the missing evidence that prevents a firmer judgment.
