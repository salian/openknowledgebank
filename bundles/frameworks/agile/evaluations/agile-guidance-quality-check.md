# Agile Guidance Quality Check

Score each criterion 0-2.

| Criterion | 0 | 1 | 2 |
| --- | --- | --- | --- |
| Source grounding | Omits source basis | Mentions Agile generally | Anchors in Manifesto values/principle themes and names missing evidence |
| Context evidence | Prescribes from labels | Uses partial context | Requires workflow, feedback, delivery, quality, and constraint evidence |
| Principle alignment | Focuses on ceremonies only | Mentions values loosely | Maps recommendation to customer value, working software, collaboration, adaptation, quality, pace, simplicity, and reflection |
| Method distinction | Equates Agile with one method | Lists methods without tradeoffs | Separates umbrella Agile from method-family mechanics and recommends by fit |
| Risk handling | Ignores downsides | Notes generic risk | Explains delivery, team, governance, compliance, and quality tradeoffs |
| Action safety | Recommends live changes without confirmation | Some caution | Requires confirmation for workflow, staffing, reporting, contract, release, or customer-facing changes |
