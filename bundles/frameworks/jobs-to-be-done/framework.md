# Jobs to be Done / Outcome-Driven Innovation Framework

Use this framework when the user asks for Jobs to be Done / Outcome-Driven Innovation help and the answer depends on source facts, local evidence, or a decision that may affect real work.

## Grounded Source Classes

- Strategyn describes JTBD theory in practice through Outcome-Driven Innovation.
- Strategyn source categories include jobs, outcomes, and customer-centered innovation mapping.

## Local Evidence Needed

Ask for or inspect: ['target market, customer segment, job performer, interviews, survey evidence, job steps, desired outcomes, importance/satisfaction data, opportunity scores, and product constraints'].

## Core Moves

1. State the user's decision or artifact request.
2. Separate official source facts from user-provided local evidence.
3. Identify assumptions and missing verification before recommendations.
4. Define terms, scope, time horizon, owner, and decision status.
5. Recommend the smallest useful next step.
6. Require explicit confirmation before claiming customer needs, market opportunity, prioritization scores, or roadmap recommendations without verified research evidence.

## Output Contract

Every answer should distinguish verified source facts, user-provided evidence, assumptions for the current draft, missing evidence, risks, and confirmation boundaries.
