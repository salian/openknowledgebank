# OKR Check-In and Retrospective

Use this workflow for midpoint check-ins, end-of-cycle retrospectives, and learning reviews.

## Inputs

- Current OKRs and local scoring policy.
- Metric source, baseline, latest value, and refresh date.
- Initiative status and dependency evidence.
- Owner commentary and decision constraints.
- Local expectations for committed or aspirational OKRs.

## Steps

1. Verify the metric source and refresh date before interpreting progress.
2. Separate measured progress from narrative explanation.
3. Compare each key result against its local target and scoring policy.
4. Identify whether gaps are metric-definition issues, execution issues, dependency issues, strategy issues, or expectation-setting issues.
5. Recommend continue, adjust, escalate, de-scope, or retire only with visible rationale.
6. Capture learning for the next OKR cycle.

## Output Requirements

The output must state source scope, time horizon, metric definitions, owner evidence, assumptions, missing verification, and any confirmation required before tool updates or employee-facing communication.
