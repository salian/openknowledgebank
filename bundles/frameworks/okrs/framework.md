# Objectives and Key Results Framework

Objectives and Key Results, or OKRs, connect a qualitative objective with measurable key results. The framework is useful when a team needs alignment, focus, and reviewable evidence of progress. It becomes unsafe or shallow when an agent invents strategy, targets, owners, scoring rules, or tool process.

## Core Moves

1. State the decision or operating problem the OKRs should support.
2. Inspect user-provided strategy, priorities, metric definitions, baseline values, ownership, cadence, and local OKR policy.
3. Draft objectives as outcomes or direction, not task lists.
4. Draft key results as measurable evidence of progress toward the objective.
5. Separate initiatives from key results: initiatives are work bets; key results are evidence.
6. Mark each OKR as committed, aspirational, or locally undefined only when the user's process supports that distinction.
7. Review tradeoffs, dependencies, metric risks, and missing evidence before recommending adoption.
8. Require explicit confirmation before changing live OKR systems, employee-facing communications, HR policy, compensation links, or executive commitments.

## Interpretation Rules

- OKRs are not a replacement for strategy; weak strategy produces weak OKRs.
- A measurable key result still needs a verified metric definition, baseline, target, and owner.
- Activity completion can be an initiative, but it is usually weak as a key result unless completion itself is the validated evidence needed.
- Committed and aspirational OKRs should not be mixed without labeling expectations and review consequences.
- Do not copy another organization's cadence, score targets, or hierarchy as if it is universal.
- Avoid mechanical links between OKR scores and compensation unless the user's reviewed policy explicitly requires them.

## Output Contract

Every OKR answer should distinguish:

- verified framework facts from public sources;
- user-provided local evidence about strategy, metrics, owners, cadence, and policy;
- assumptions for the current draft;
- missing verification before conclusion, adoption, or live change.
