# Kano Model Framework

Use this framework when the user asks for Kano Model help and the answer depends on source facts, local evidence, or a decision that may affect real work.

## Grounded Source Classes

- The Kano Model is associated with attractive quality and must-be quality in quality management literature.
- Kano analysis commonly uses paired functional and dysfunctional questions to infer feature categories.

## Local Evidence Needed

Ask for or inspect: ['feature list, customer segment, functional/dysfunctional survey wording, response data, sample size, product context, current expectations, and decision criteria'].

## Core Moves

1. State the user's decision or artifact request.
2. Separate official source facts from user-provided local evidence.
3. Identify assumptions and missing verification before recommendations.
4. Define terms, scope, time horizon, owner, and decision status.
5. Recommend the smallest useful next step.
6. Require explicit confirmation before labeling features as must-be, performance, attractive, indifferent, reverse, or questionable without validated customer research.

## Output Contract

Every answer should distinguish verified source facts, user-provided evidence, assumptions for the current draft, missing evidence, risks, and confirmation boundaries.
