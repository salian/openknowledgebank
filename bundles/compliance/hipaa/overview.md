# Overview

HIPAA is a U.S. healthcare compliance framework covering several related Administrative Simplification rule families. The bundle treats HHS OCR and CMS pages as authoritative public guidance categories, current CFR text as the source for regulatory structure, and user-provided evidence as necessary for organization-specific conclusions.

## What This Bundle Is For

- Triage whether a HIPAA question is about privacy, security, breach notification, enforcement, transactions, identifiers, code sets, or a related rule family.
- Identify the official source categories that must be checked before answering.
- Produce source-aware briefs that separate verified facts from assumptions and missing evidence.
- Route deeper rule-family work to related bundles or current official sources.

## What This Bundle Is Not For

- It is not legal advice.
- It is not a complete legal commentary.
- It does not decide compliance, breach reportability, or enforcement exposure from incomplete facts.
- It does not replace state law, 42 CFR Part 2 analysis, contract review, security engineering, incident response, or professional review.

## Source Discipline

Every HIPAA output should distinguish:

- verified official-source facts;
- HHS OCR, CMS, eCFR, or Federal Register source categories checked for the specific task;
- facts provided by the user;
- assumptions made only for a draft;
- missing facts or sources that block a conclusion.

## Risk Boundary

Submitting breach reports, contacting HHS OCR or another authority, sending notices, disclosing PHI/ePHI, changing BAAs, changing policies, changing security controls, or changing transaction processing are risky actions. An agent should require explicit user confirmation and professional review before those actions.
