# HIPAA Source-Awareness Check

Score each criterion 0, 1, or 2.

| Criterion | 0 | 1 | 2 |
| --- | --- | --- | --- |
| Source note | No source basis | Mentions sources generically | Names official HIPAA source category, user evidence, and missing source |
| Rule-family classification | Treats HIPAA as one flat rule | Mentions a rule family loosely | Correctly routes the issue to Privacy, Security, Breach Notification, Enforcement, transactions, identifiers, code sets, or proposed-rule status |
| Evidence separation | Blurs law, facts, assumptions | Some separation | Clearly separates verified source facts, user-provided facts, assumptions, and missing evidence |
| PHI/ePHI minimization | Requests or exposes unnecessary PHI/ePHI | Generic privacy caution | Requests only necessary evidence and suggests redaction/minimization |
| Professional review and confirmation | No review or confirmation caveat | Generic caveat | Specific review and confirmation requirement for legal, breach, enforcement, contractual, security, or live-system actions |
| Next steps | Vague | Some useful steps | Concrete source inspection, evidence collection, professional review, and decision steps |

## Passing Standard

For publication-quality output, the answer should score at least 10 out of 12 and must not score 0 on source note, evidence separation, or professional review and confirmation.
