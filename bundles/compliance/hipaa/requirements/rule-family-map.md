# HIPAA Rule-Family Map

Use this as a routing map, not as a substitute for the current legal text or professional review.

| Question Class | Primary Source Category | Evidence Needed Before Conclusion |
| --- | --- | --- |
| Privacy, uses, disclosures, individual rights, notices, minimum necessary, or PHI handling | HHS OCR Privacy Rule pages and 45 CFR Part 164 Subpart E | Entity role, data context, use/disclosure purpose, user-provided documents, and current source inspection. |
| ePHI safeguards, risk analysis, access controls, audit logs, incident handling, or cybersecurity | HHS OCR Security Rule pages and 45 CFR Part 164 Subpart C | System inventory, ePHI flows, risk analysis, controls, policies, and current source inspection. |
| Breach notification, unsecured PHI, Secretary notice, individual notice, media notice, or business-associate incident reporting | HHS OCR Breach Notification pages and 45 CFR 164.400-414 | Incident facts, affected records, acquisition/use/disclosure facts, risk assessment, timing, contracts, and professional review. |
| OCR investigations, compliance reviews, civil money penalties, or cooperation with OCR | HHS OCR enforcement pages and 45 CFR Part 160 | Notice/request text, investigation facts, records requested, legal counsel input, and current source inspection. |
| Electronic transactions, eligibility, claim status, claims, remittance, EFT, or claims attachments | CMS/HHS Administrative Simplification sources and 45 CFR Part 162 | Transaction type, parties, implementation guide, standard version, payer/provider context, and test evidence. |
| NPI, EIN, code sets, or other identifiers/standards | CMS/HHS Administrative Simplification sources and 45 CFR Part 162 | Identifier type, entity role, transaction context, code-set/version source, and current source inspection. |
| Proposed cybersecurity changes to the Security Rule | HHS OCR NPRM and Federal Register sources | Current final/proposed status, effective dates, and legal review before treating as binding. |

## Drafting Rule

When a user asks for a HIPAA answer, first classify the rule family. Then name the official source category and the missing user evidence before offering a conclusion. If the exact rule, deadline, form field, implementation detail, or system control has not been checked, mark it as needing verification.
