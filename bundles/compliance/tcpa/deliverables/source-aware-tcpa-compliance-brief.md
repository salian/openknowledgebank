# Source-Aware TCPA Compliance Brief

Use this format for TCPA-sensitive questions. It is a brief for review, not legal advice or operational approval.

## Required Sections

### Direct Answer

State the answer in one short paragraph. If evidence is incomplete, say that the scenario is not ready for a compliance conclusion and name the blocking gaps.

### Source Note

Name:

- official FCC, eCFR, statutory, FTC, state, sector, platform, or contract source categories checked;
- user-provided outreach, consent, revocation, DNC, vendor, platform, or campaign evidence used;
- sources or facts still missing.

### Outreach Scope

Identify channel, purpose, sender/caller/texter/fax sender, seller, recipient type, geography, number type when known, technology, and vendor involvement.

### Evidence Matrix

| Classification | Evidence |
| --- | --- |
| Verified from official source |  |
| Provided by user |  |
| Assumed for this draft |  |
| Needs verification |  |

### Source-Grounded Issues

List the issues that require current source inspection, such as consent, revocation, do-not-call screening, caller identity, fax advertisement rules, exemption claims, and related TSR or state-law checks.

### Risk Boundary

State that the brief is not legal advice and should be reviewed by a qualified legal or compliance professional before campaign, calling, texting, faxing, enforcement, contractual, or regulatory decisions.

### Next Checks

List concrete checks needed before any risky action. Include official source inspection, consent record review, revocation and suppression tests, DNC process verification, vendor/platform evidence, and adjacent state or sector checks when relevant.

## Quality Bar

A good brief refuses to turn a purchased list, lead form, prior customer relationship, vendor assurance, or missing opt-out record into a compliance conclusion without source inspection and user evidence.
