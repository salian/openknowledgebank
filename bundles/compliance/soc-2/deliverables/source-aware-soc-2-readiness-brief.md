# Source-Aware SOC 2 Readiness Brief

## Required Sections

### Direct Answer

Answer the user's question at the highest supportable confidence level. If a conclusion depends on current AICPA material, auditor judgment, or user evidence that is missing, say that plainly.

### Source Note

Name:

- official AICPA/CIMA source category checked or still needed;
- user-provided report, auditor request, system description, control inventory, or evidence used;
- missing source or evidence that prevents a final conclusion.

### Scope and Category Framing

State the service, system boundary, report type or readiness question, trust services category context, customer requirement, and time period, if known.

### Evidence Table

| Item | Status | Source or Owner | Why It Matters |
| --- | --- | --- | --- |
| Official source category | verified / missing |  |  |
| Auditor request or engagement context | verified / missing |  |  |
| System description | verified / missing |  |  |
| Control inventory | verified / missing |  |  |
| Evidence exports or artifacts | verified / missing |  |  |
| Report period or readiness period | verified / missing |  |  |

### Analysis

Explain what can be said from verified facts and user-provided evidence. Keep criteria, point-of-focus, control-mapping, and opinion-language details out unless the user has supplied an authorized source excerpt or the auditor has provided the specific request.

### Risks and Professional Review

List reliance risks and name the reviewer needed: CPA/service auditor, legal counsel, security owner, compliance owner, product owner, or customer-response owner.

### Next Actions

Request the narrowest useful next evidence. Require explicit confirmation before contacting auditors, sending customer responses, exporting evidence, changing controls, or modifying production systems.
