# Source-Aware Triage

## Inputs

- User request and intended decision.
- Official or authoritative source category already inspected, if any.
- Local documents, system records, policies, contracts, logs, reports, filings, forms, or other evidence.
- Missing facts that block reliance.

## Steps

1. Classify the request and identify the governing source category.
2. Ask for only minimum necessary evidence.
3. Separate verified facts, user-provided facts, assumptions, and missing evidence.
4. Draft the [source-aware brief](../deliverables/source-aware-brief.md).
5. Mark qualified review and explicit confirmation requirements for risky actions.

## Output Contract

Include direct answer, source note, evidence matrix, missing evidence, risk boundary, and next steps.
