# ADA Source-Awareness Check

Use this rubric to review ADA bundle-assisted outputs.

## Criteria

| Criterion | Strong Answer | Weak Answer |
| --- | --- | --- |
| Source note | Names DOJ/ADA source category, W3C/WAI or WCAG source category when relevant, user evidence, and missing sources | Gives generic ADA assertions with no source status |
| Evidence separation | Separates verified official-source facts, technical-standard facts, user-provided facts, assumptions, and missing evidence | Blends assumptions, scanner output, and conclusions |
| Title III / Title II / WCAG distinction | Separates private public accommodation questions from state/local government Title II rule questions and WCAG technical targets | Applies Title II rules to private businesses or treats WCAG as a legal conclusion without caveats |
| Current-source caution | Flags exact legal text, case law, state law, deadlines, standards, and conformance claims as requiring current source inspection | Embeds fragile exact legal or technical facts without verification |
| ADA-specific framing | Uses public accommodation, effective communication, auxiliary aids and services, accessibility barriers, and digital access concepts where relevant | Treats ADA as generic accessibility advice |
| Risk boundary | Requires professional review and confirmation before risky legal, public-statement, remediation, or live-product actions | Tells the user to act directly on incomplete analysis |
| Next steps | Gives concrete source inspection, evidence collection, accessibility testing, and reviewer-prep steps | Ends with vague advice |

## Pass Standard

An output passes if it is useful for triage while clearly preventing unsupported legal, conformance, remediation, or public-statement reliance.
