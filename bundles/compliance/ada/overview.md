# Overview

ADA is the common shorthand for the Americans with Disabilities Act. This bundle focuses on Title III public accommodation and digital accessibility questions. It treats DOJ and ADA.gov materials as official ADA source categories, W3C/WAI as the technical WCAG source category, and user-provided evidence as necessary for any organization-specific or product-specific conclusion.

## What This Bundle Is For

- Triage whether a question concerns ADA Title III public accommodations, digital accessibility, effective communication, auxiliary aids and services, WCAG technical references, public statements, vendor evidence, remediation planning, or professional review.
- Identify official source categories that must be checked before answering.
- Distinguish private-sector Title III questions from the DOJ Title II web/mobile rule for state and local governments.
- Produce source-aware briefs that separate verified source facts from user evidence, assumptions, and missing verification.

## What This Bundle Is Not For

- It is not legal advice.
- It is not an accessibility audit or WCAG certification.
- It is not litigation strategy or a substitute for current case-law research.
- It does not decide ADA compliance from incomplete facts, scanner output, overlays, or unreviewed remediation claims.
- It does not replace current DOJ, ADA, W3C/WAI, state-law, case-law, product, or professional source inspection.

## Source Discipline

Every ADA output should distinguish:

- verified DOJ, ADA, or W3C/WAI source facts;
- the exact source category inspected for the specific task;
- facts provided by the user about the business, website, app, content, audit, testing, policy, vendor, communication, or remediation state;
- assumptions made only for a draft;
- missing facts or sources that block a legal, conformance, remediation, public-statement, or reliance conclusion.

## Risk Boundary

Publishing accessibility statements, claiming ADA compliance, claiming WCAG conformance, responding to legal demands, contacting regulators, changing live products, committing remediation timelines, exporting user data, or sending legal/compliance communications are risky actions. An agent should require explicit user confirmation and qualified professional review before those actions.
