# Roadmap

OpenKnowledgeBank is being built as a free, open, inspectable, and remixable library of knowledge bundles for AI agents.

## Phase 0: Foundation

- Seed the repository structure.
- Define bundle organization for roles, industries, capabilities, tools, frameworks, compliance, jurisdictions, deliverables, and datasets.
- Add starter registry files.
- Add contribution and licensing docs.
- Create the first seed bundle skeleton.

## Release Philosophy

OpenKnowledgeBank should stay in public preview until there is enough breadth to justify calling it a release.

- Do not tag a formal `v1`-style release while the catalog is still a small proof set.
- Treat 100 useful bundles as the first meaningful public release threshold.
- Build toward a 2,000-bundle open library across roles, tools, frameworks, compliance regimes, jurisdictions, deliverables, datasets, capabilities, and industries.
- Prefer validated, source-backed, agent-usable bundles over hitting a bundle count with thin content.

## Phase 1: Public Preview And Validation

- Create `ga4-analytics-specialist` as the first full production benchmark bundle.
- Keep `performance-marketer` as a seed skeleton until after the benchmark.
- Add CI validation for bundle structure, registry metadata, internal links, and public-safety checks.
- Add contributor templates for bundle suggestions, issues, and pull requests.
- Add several more bundles, including non-role bundles where useful.
- Add examples that show how to use bundles with agents.
- Add validation tooling for bundle structure and metadata.
- Improve the registry so tools and websites can consume it.
- Publish clear contribution guidance.

## Phase 2: 100-Bundle Public Release Candidate

- Reach 100 useful, validated bundles before a formal release.
- Add bundle status levels such as draft, review, and stable.
- Add source coverage and validation indicators.
- Add more realistic examples and workflows.
- Add packaging automation for downloadable bundle archives without calling it a formal release too early.
- Improve docs for bundle authors and agent builders.
- Add contributor metadata and public contributor profile support.
- Include measured scorecards for priority bundles where feasible.

## Phase 3: 500-Bundle Catalog Expansion

- Support external OpenKnowledge repositories in the registry.
- Add contributor recognition.
- Add reference integrations.
- Grow the catalog around high-value roles, industries, capabilities, tools, frameworks, compliance regimes, jurisdictions, deliverables, and datasets.
- Add search facets that help agents and humans discover adjacent bundles.
- Expand benchmark coverage across multiple domains, not only analytics.

## Phase 4: 2,000-Bundle Open Library

- Build a broad, navigable corpus with strong metadata and consistent quality gates.
- Maintain contributor profile pages and public credit.
- Use popularity, download counts, validation status, and measured evaluation status as discovery signals.
- Keep trusted bundles in the public repo; support community and unverified external bundles with clear labels.
- Defer commercial surfaces until the free/open library has real usage and reputation value.
